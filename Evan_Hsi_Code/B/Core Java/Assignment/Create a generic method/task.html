Create a generic method

Polymorphism is one of the core pillars of object-oriented programming. It allows developers to reuse logic in various scenarios in order to reduce the overall number of lines of code.

For instance, if you need to sort some things, why would you implement a different class for each and every type of thing you need to sort? Is the logic of sorting that much different from thing to thing? Isn't there a way that you can streamline this so that a single class can handle all cases?

Well, in fact, there is: generics. Generics were introduced in Java 5 and are most commonly associated with the Collections API. This is appropriate since prior to Java 5, working with raw collections was cumbersome and error-prone. The introduction of this feature was a boon for Java developers, and greatly appreciated.

Your task is to provide an implementation of a generic method, one that can work with any type and print out all of the items of a provided array (which could be an array of ints, Strings, etc.). Do not use overloaded methods, as this will not pass.

Once you have implemented a solution, you can hit the Check button below to run your implementation against some unit tests provided by your trainer.
