Reading input from the console (with BufferedReader)

As we saw in the first task, printing text to the console is pretty straightforward - especially with the System class at our disposal. Next, we move on to reading from the console.

In order to accomplish this task you will again be making use of the System class, but you will also need to use the BufferedReader member included in the Task class declaration to allow you to read from an input stream (in this case, the console). Provide an implementation that will prompt the user for input and return their response.

Once you have implemented a solution, you can hit the Check button below to run your implementation against some unit tests provided by your trainer.

Pay close attention to the arguments of the BufferedReader's constructor. Notice that it needs a Reader implementation passed to it. There is an implementation of this abstract class that will help you to read from the system's input stream (console).

Most of the methods within the BufferedReader's API throws checked exceptions, so you will be required handle these within your implementation.
